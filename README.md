# Engineering Agency OS — Antigravity Workspace

This workspace is prepared so Antigravity or Claude can work in a staged, controlled way.

## What this workspace already contains
- `.agents/` guidance for Antigravity
- `docs/prd/` product and build rules
- `docs/stages/` stage-by-stage briefings and prompts
- placeholder `app/`, `components/`, `lib/`, `types/`, `supabase/` directories
- `.env.example` for environment variables
- starter project notes so the AI does not invent structure randomly

## How to use
1. Open this folder as a dedicated workspace in Antigravity.
2. Read `COPY_ME_FIRST.md`.
3. Use the files in `docs/prd/` as the source of truth.
4. Start with `docs/stages/stage-01/STAGE_01_PROMPT_ANTIGRAVITY_CLAUDE.md`.
5. Never run more than one stage at a time.

## Build philosophy
- desktop-first
- operational, not flashy
- table-first UX
- calm neutral palette
- tight MVP scope
- file metadata in app, actual engineering files in Google Drive
